## 1.1 — `catalog.json` schema (final v1.0)

Updated from Slice 0.4 with `llm_hint`, content-hash URLs, and metadata for downstream consumers. **This is the contract Sonnet should land in Slice 0.4.**

```json
{
  "schema_version": "1.0",
  "registry": "official",
  "registry_url": "https://agents.clawwidget.com",
  "generated_at": "2026-04-16T12:00:00Z",
  "generator": {
    "name": "cwidget-catalog-builder",
    "version": "0.1.0",
    "commit": "<git-sha-of-source-repo>"
  },
  "skill_count": 5,
  "skills": [
    {
      "slug": "official/media/youtube-dl",
      "name": "youtube-dl",
      "description": "Downloads YouTube audio and saves to Desktop.",
      "llm_hint": "Use when the user pastes a YouTube URL and wants the audio locally. Returns { saved_path: string, duration_ms: integer }. Compose with `official/media/transcribe-audio` to get a transcript chain. Do not use for playlists — single video only in v1.",
      "trigger": "on-clipboard",
      "composable": true,
      "remote_expose": false,
      "human_in_loop": false,
      "sidecars_required": [],
      "requires": [],
      "raw_url": "https://agents.clawwidget.com/skills/official/media/youtube-dl/sha256-3f7a9b…/skill.md",
      "canonical_url": "https://agents.clawwidget.com/skills/official/media/youtube-dl/skill.md",
      "hash": "sha256:3f7a9b2e…",
      "size_bytes": 4827,
      "added_at": "2026-04-10T08:00:00Z",
      "last_modified_at": "2026-04-15T16:22:00Z",
      "category": "media",
      "tags": ["youtube", "audio", "download"]
    }
  ]
}
```

**Key shape decisions:**

- **`llm_hint`** — the Anthropic principle 1 surface. Rich prose, written for AI consumers, ~200–800 chars typical. NOT shown to humans (UI uses `description`).
- **`raw_url` is content-hashed.** `/skills/<registry>/<category>/<name>/sha256-<hex>/skill.md` (three path segments after `skills/` for official entries — CP004). This is the URL the runtime fetches when installing. **Cache-Control: immutable, max-age=31536000.** Hash collision is impossible by construction; cache invalidation is automatic on update. Solves the CDN-cache problem completely.
- **`canonical_url`** — unhashed pretty URL for humans browsing, GitHub README links, and "Custom/Modified" badge cross-references. Cache-Control short (60s). Runtime never fetches this.
- **`size_bytes`** — lets Slice 2 do an early reject if a skill is suspiciously large before downloading.
- **`category` + `tags`** — feed Slice 3 dynamic filtering (Anthropic principle 3). Free-form for v1; tightenable later via contract.
- **`generator.commit`** — provenance from source repo to catalog snapshot. Lets us trace any catalog entry back to the exact GH commit that produced it.
- **Backward/forward compatibility:** `schema_version: "1.0"`. Runtime parser uses `#[serde(deny_unknown_fields = false)]` so future field additions don't break old binaries. New required fields require schema version bump.

## 1.5 — GitHub Action: `validate.yml` (PR check)

Runs on every PR touching `skills/**`.

```yaml
name: Validate Catalog
on:
  pull_request:
    paths: ['skills/**', '.github/workflows/validate.yml']

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install cwidget
        run: |
          # Pinned to a known-good cwidget release with --catalog-mode support
          curl -sSL https://releases.clawwidget.com/cwidget-${CWIDGET_VERSION}-linux-x64.tar.gz | tar -xz
          install -m 755 cwidget /usr/local/bin/
      - name: Validate every skill against catalog rules
        run: cwidget validate --catalog-mode skills/
      - name: Verify catalog graph is flat + closed
        run: cwidget validate --catalog-graph skills/
```

`--catalog-mode` invokes the publish-time rules from Slice 0.7 (`RULE_CATALOG_PUBLISH_REQUIRES_TRUSTED_ONLY`, `RULE_CATALOG_PUBLISH_REQUIRES_FLAT`, etc.). `--catalog-graph` is a separate validation pass that walks every skill's `requires:` and confirms (a) every referenced slug exists in the same catalog directory, (b) no cycles, (c) flat-only.

**PR fails fast and loud** on any rule_id from Slice 0. PR template tells contributors exactly what each error means (lifted from `help_doctor_explain.json`).

## 1.6 — GitHub Action: `publish.yml` (build + deploy)

Runs on push to `main` (post-merge).

```yaml
name: Publish Catalog
on:
  push:
    branches: [main]
    paths: ['skills/**']

jobs:
  publish:
    runs-on: ubuntu-latest
    permissions:
      contents: write   # for committing generated catalog
      pages: write      # if using GitHub Pages
    steps:
      - uses: actions/checkout@v4
      - name: Install cwidget
        run: <same as validate.yml>
      - name: Build catalog.json + manifest + content-hashed skill copies
        run: |
          cwidget dev build-catalog \
            --source skills/ \
            --out dist/ \
            --registry official \
            --registry-url https://agents.clawwidget.com \
            --commit ${{ github.sha }}
      - name: Deploy to Cloudflare Pages
        uses: cloudflare/pages-action@v1
        with:
          apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}
          accountId: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
          projectName: official-skills
          directory: dist/
```

**`cwidget dev build-catalog`** is a new dev subcommand (small Rust addition — sits next to `cwidget dev export-contracts`). Outputs:

```
dist/
├── catalog.json
├── .well-known/
│   └── mcp-registry.json
└── skills/
    └── official/
        ├── media/
        │   ├── youtube-dl/
        │   │   ├── skill.md                        # canonical_url target
        │   │   └── sha256-3f7a9b…/skill.md         # raw_url target (immutable)
        │   └── transcribe-audio/
        │       ├── skill.md
        │       └── sha256-…/skill.md
        └── ...
```

The hashed-directory approach means every skill version is permanently addressable. Old versions stay reachable until the GitHub repo's git history is purged. **No silent updates to installed users — Fork on Edit holds.**

## 1.7 — CDN: Cloudflare Pages binding

| Setting | Value |
|---|---|
| Domain | `agents.clawwidget.com` |
| TLS | Strict (Full Strict mode); HSTS preload |
| HTTP fallback | **Disabled.** All HTTP requests 308 → HTTPS. |
| Cache (`/catalog.json`) | `Cache-Control: public, max-age=60, stale-while-revalidate=300` |
| Cache (`/skills/**/sha256-*/skill.md`) | `Cache-Control: public, max-age=31536000, immutable` |

## 1.9 — Operational runbook (one-pager in repo)

- **Publish a new skill:** Open PR with `.md` file under `skills/official/<category>/<slug>.md`. CI validates. CODEOWNER approves. Merge. Catalog auto-deploys within 2 minutes.
- **Update a skill:** Same. Old hashed URL keeps working forever (Fork on Edit on the user side; cache stays valid). New `catalog.json` points at new hash; new installs get new content; existing installs untouched.
- **Retract a skill (rare):** Remove from `skills/`. New catalog omits it. Existing local installs **remain functional** (Fork on Edit — file is theirs now). New installs fail with `JIT_DEPENDENCY_NOT_IN_CATALOG`. **There is no kill-switch in MVP.** This is intentional: local-first means we don't have a remote off-button. If a critical security issue ships, the response is (a) ship a `cwidget` binary update that warns on the bad slug, (b) engage users via app update + release notes. Document this clearly in the runbook so we're not surprised later.
- **Domain compromise (worst case):** Cloudflare account takeover or DNS hijack. Mitigations: 2FA-required on GH + Cloudflare; CAA records; HSTS preload prevents mass MITM. Detection: external uptime + content-hash monitoring (a tiny GH Action that fetches `agents.clawwidget.com/catalog.json` daily and verifies hash against repo). Recovery: revoke certs, restore from git.
